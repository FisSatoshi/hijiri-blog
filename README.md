# ヒジリBlog

自己紹介・学歴経歴・日々のつぶやき・カテゴリ別記事（ゲーム／お出かけ／その他）を持つ、
個人ブログのひな形一式です。GitHub Pagesで無料公開し、専用の管理画面からブラウザだけで
記事の投稿・編集・写真や動画の追加・プロフィールの編集ができます。

```
hijiri-blog/
├── index.html      … トップページ（自己紹介・経歴・最新記事）
├── blog.html        … 記事一覧（カテゴリ絞り込み付き）
├── post.html         … 記事の詳細ページ
├── admin.html        … 管理画面（合言葉ロック）
├── css/style.css      … デザイン
├── js/site.js          … 一般ページの表示ロジック
├── js/github-api.js     … GitHubへの読み書き
├── js/admin.js            … 管理画面のロジック
└── data/
    ├── posts.json          … 全記事データ（管理画面が自動で書き換える）
    └── profile.json         … 自己紹介・経歴データ（管理画面が自動で書き換える）
```

---

## 1. 公開までの手順（最初の1回だけ）

1. **GitHubにリポジトリを作る**
   GitHub上で新しいリポジトリを作成します（例：`hijiri-blog`）。公開設定は
   **Public**にしてください（無料のGitHub Pagesは基本的にPublicリポジトリ向けです）。

2. **このフォルダの中身をリポジトリにアップロードする**
   - GitHubのWeb画面で「Add file → Upload files」からこのフォルダの中身一式を
     ドラッグ＆ドロップするか、
   - お使いのPCにGitが入っていれば、通常の`git init` → `git add` → `git commit` →
     `git push`でも構いません。

3. **GitHub Pagesを有効にする**
   リポジトリの `Settings → Pages` を開き、
   `Source` を `Deploy from a branch`、ブランチを `main`（フォルダは `/root`）に設定して保存します。
   数分待つと `https://（あなたのユーザー名）.github.io/（リポジトリ名）/` で公開されます。

4. **個人アクセストークン（PAT）を発行する**
   管理画面から記事を保存するために必要です。
   - GitHubの `Settings → Developer settings → Personal access tokens → Fine-grained tokens` を開く
   - 「Generate new token」で、対象リポジトリを**このブログのリポジトリだけ**に限定
   - `Repository permissions → Contents` を **Read and write** に設定して発行
   - 表示されたトークン（`github_pat_...`）をコピーしておく（この画面を閉じると二度と見られないので注意）

5. **管理画面にログインして接続設定をする**
   `https://（あなたのユーザー名）.github.io/（リポジトリ名）/admin.html` を開く
   - 初回はこのブラウザだけで使う「合言葉」を決める
   - 「GitHub連携設定」タブで、ユーザー名・リポジトリ名・ブランチ名(`main`)・トークンを入力して保存
   - 「接続テスト」を押して成功すればOK

これで準備完了です。以後は `admin.html` から記事を書けば、数十秒〜数分後に
サイトへ反映されます（GitHub Pagesがビルドし直す時間がかかるため、即時ではありません）。

---

## 2. 日常の使い方

- **記事を書く**：admin.html →「✏️ 記事を書く」タブ。タイトル・カテゴリ・日付・抜粋を入力し、
  本文はツールバーで太字・見出し・箇条書き・引用・リンクなどの装飾ができます。
  写真アイコン🖼️・動画アイコン🎬から、その場でファイルをアップロードして本文に挿入できます
  （＝あとから写真や動画を追加していく使い方が可能です）。「公開する」を押すと保存されます。
- **記事を編集・削除する**：「🗂 記事の管理」タブから一覧を見て、編集/削除ボタンを押します。
- **プロフィールを直す**：「🙋 プロフィール編集」タブで自己紹介文と学歴・経歴のタイムラインを
  編集できます（項目は自由に追加・削除できます）。

---

## 3. 「自分しか編集できない」ことについて

- サイトのコード自体（HTML/CSS/JS）は誰でも見られる状態です（GitHub Pagesの仕組み上そうなります）。
- しかし、**実際にデータを書き換えるにはあなたのGitHub個人アクセストークンが必要**です。
  このトークンは他人には教えず、あなたの端末のブラウザにだけ保存されるので、
  トークンを持たない第三者はデータを書き換えられません。
- 管理画面に入るための「合言葉」は、共有パソコンなどで画面を覗き見されないための
  簡易ロックです。本当の防御はトークンの方だと考えてください。
- より強く守りたい場合は、リポジトリをPrivateにし、GitHub ProプランでPrivateリポジトリの
  Pagesを有効化する方法もあります（この場合は少し設定が変わります）。

---

## 4. 制限事項・補足

- 画像・動画のアップロードはGitHub APIの仕組み上、**1ファイルおおよそ1〜数MB程度まで**が
  安定して動く目安です（動画は特にサイズが大きくなりがちなので、圧縮するか、
  YouTubeの限定公開などにアップロードしてリンクを貼る方法もおすすめです）。
- 記事や写真はすべて `data/posts.json` と `assets/` フォルダにテキスト・ファイルとして
  保存されます。中身はいつでもGitHub上で直接確認・編集もできます。
- カテゴリ（日記／ゲーム／お出かけ／その他）を増やしたい場合は、
  `js/site.js` と `admin.html` / `js/admin.js` 内の `CATEGORIES` 関連箇所と
  `css/style.css` の `.tag.xxx` を増やせば対応できます。
